Hi guys.

I redesigned a few stuff in the core about music elements manipulation
in CAStaff for version 0.5. I planned this change for a long time in my
head but postponed it a while. Updates mainly include stability fixes
and robustness of the playable/non-playable elements in the staff.

Among some function renamings and usability fixes, the biggest addition
is CAStaff::synchronizeVoices(). This function takes care of the staff
consistency - voices share non-playable elements (barlines, clefs, time
signatures etc.). The main question until now was: What happens if I
insert a note or rest in front of such a sign. How are other voices
effected? What happens if I put a barline right behind a half note and
there is a whole note in the second voice? What if there are two
barlines with different timeStarts and nothing (no rests) inbetween? All
these issues should now be fixed by calling synchronizeVoices() method.
