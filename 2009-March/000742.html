Hi,

A follow up to some discussion concerning Javascript integration
to Canorus. I wanted to avoid bad stress from the beginning.

While I agree that both Python and QtScript have Qt support I
think it's agood idea that the interface does not have any
direkt Qt support. Keeping a plugin interface clean of any
framework things makes life later a lot easier, like getting
old plugins to run where the developer disappeared.

I disagree to only having one language but I'm very well
aware that there can be much work involved if any scripting
language dependencies get into the interface itself. Though
SWIG might not be the easiest way to go for the interface it's
still currently one of the easiest for multiple languages.

That's one of the reasons why the JavaScript approach
currently - it's completely different. For script developers
requiring user interface interaction I think JS is currently
not the way to go generally. QtScript is too much dependant
on Qt itself and other dialects might not even known any
user interface creation interfaces (I haven't investigated
here hence I even don't know the language itself really).

So for the forseeable future I'd say Python and maybe one
other language (like Ruby). When we have more developers
and/or when SWIG or other scripting application interfaces get
more easily (i.e. more automation usually) we might expand on
more languages as well.

What are your thoughts ?
