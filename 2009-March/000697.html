Fonts never showed under macosx. I would always get a square box,  
which according to Qt means that something went wrong (possibly  
character not available in font).

After creating a QFont I tested creating a QFontInfo from it and  
querying the family. The font of the loaded family was Helvetica, even  
though in CACanorus::init the loading of Emmentaler-14.ttf was  
returning a valid ID.

Looks like for some reason the font lookup under MacOSX is more picky  
and only works if you ask for "Emmentaler 14" instead of just  
"Emmentaler". I'm attaching a path for this, though it would be better  
if you created a define globally with the string "Emmentaler 14" and  
used the define instead of typing it again and again all over the place.

Now that fonts work I tried to load an xml example and got garbage for  
notes. Any ideas about this?
