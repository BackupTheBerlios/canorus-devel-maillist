Grzegorz Adam Hankiewicz pravi:
> The following patch modifies the build defines to avoid looking for
> resources in the typical /Applications/Canorus.app directory to
> instead look in a relative Canorus.app directory.
>
> The result is that the .app does not have to be installed with admin
> rights and can be run from wherever you please.
>
> Thanks to this and the recent changes of moving all resources into the
> bundle, building a dmg file seems unnecessary. Usually these include a
> special background and/or installation package/instructions. However,
> more and more simple applications just pass a zip where after
> decompression the app can be run directly.
