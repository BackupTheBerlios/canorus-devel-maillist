Hi,

in my opinion, keyboard shortcuts for the recent file lists are good to
have. At the moment it always takes extra time to switch to the mouse when
opening a recent file (due to some bugs it commonly is necessary to close
and re-open a file while editing).
A possible solution for the problem of the range changing often would be to
have a shortcut to display the list, from which the user then can choose
e.g. with the number keys. Opening the most recent file could be e.g. done
with 'alt f' -> 'r' -> '1' or something like 'strg-r' -> '1'.

Best regards

Andreas Schneider
