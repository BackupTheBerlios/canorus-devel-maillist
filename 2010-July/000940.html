Hi,

I dig out a very old topic, but it's important right now as I have
plans for improving our debug log quite a lot.

I never really aversed to switch, now I feel that we have to do
the switch soon. I can't remember Georg opposing, so as long as
we don't change possibly included GPLv2 libraries let's do the step.

The reason for my rush is that I want to include a library called
log4qt (for a much improved log output and easier maintanence
of the log in general). Unfortunately it uses Apache license v2.0
which is only compatible with GPLv3.

Also I'd like to know if we want to publish the Canorus library
with LGPL(v3) or if we want to stay with GPLv3. I don't know
how strong everybody's interest is that commercial programs
might use Canorus features (like loading a Canorus score or
using the Lilypond backend). Personally I would - at least for
some time - be less restrictive to the Library only contrary
to the Canorus application itself.

Regards,

Reinhard Katzmann

Am 14.11.2007 10:58, schrieb canorus-devel at lists.berlios.de:
> I've been thinking about relicensing Canorus from GPLv2 to GPLv3, but
> I'm not an attorney and dunno much about licensing stuff. This post
> should open a discussion about the new license. All contributors and
> developers should agree to the adoption of the new license, if we
> actually decide to relicense Canorus to a newer version of GPL.
> 
> My 2 points of view:
> + GPL3 addresses patent issues. Something like our mod999el of score
> representation and GUI could be victim of patents in the future. We
> should be safer when using GPL3. GPL2 doesn't address patents at all.
> - There are lesser restrictions of publishing the source code along the
> application over internet. I see greater risk for someone to stole our
> code and reimplement it in their product. GPLv2 is pretty strict about
> combining free/non-free software.
> 
> I didn't have time to read through and completely understand the actual
> GPL3 license so I ask for your opinions.
