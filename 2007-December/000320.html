> I made the following plan for Canorus 0.5:
> - 2nd January: feature freeze - translators should start translating
> - 14th January: release candidate, extensive bug chasing, translations
> submitted
> - 20th/21th January: 0.5 release.

Agreed, time frame should be sufficient for us. I'd even put a beta out
already around christmas - to get some feedback from the "outer" world.

> Major things done:
> - staff consistency, robustness, stability
> - threaded import/export
> - import/exported opened to scripting engine
> 
> Major things to be done:
> - dynamics, tempo changes
> - printing (Reinhard?)
> - viewports reorganization (Itay?)

Printing is up to me, and I should be able to have it ready until then.
I also hope to get at least some preview too, but that depends how well
everything will work.

One not so small point: I found no proper way to integrate printing control
into Canorus. While the mainwindow is ok for starting the print action it is
not the right class to "store" all these control classes. After some analyses
I came to the conclusion that we need a base canorus control class, splitted
from the mainwindow class. I'll write more details in a separate mail. This
though it unfortunately will cost quite some time is a crucial step before
we can add more major features.

> Other stuff, if we have time:
> - staccato, marcato, portato etc. on notes
> - fermata
> - arpeggio
> - octaviation
> - pedal markings
> 
> New translations:
> - French
> - Danish
> - Norwegian
> - German
> - Traditional Chinese
> 

I updated the roadmap too last week but before this mail, maybe you
could correct it if there are any things missing or already implemented
in 0.4.
