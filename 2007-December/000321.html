Hi there!

I have been checking out latest Canorus SVN (633).

There is some weird thing going on with ledger lines. Just place a
c''' and a d''' side to side and you'll see what I mean. I took a look
at the code thinking of sending some patch, since this seemed an easy
one. When I saw the layout code I thought of more fundamental
questions.

IMO what would make sense here would be a stylesheet approach. I would
take a leaf of Qt widget system and bundle the look (representation,
view) and feel (behaviour, controller) in a style class, similar to
QStyle.

Regarding rounding issues, QPainter admits floating coordinates. Did
you give it a try?

If at some point you are planning to implement different size staves
in the same score (ossias, accompaniment parts) things like note
drawing should take this into account.
