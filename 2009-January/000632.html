Hi Nils.

The playback is done using RtMidi only and Canorus is completely
independent. You can see details here:
http://svn.berlios.de/wsvn/canorus/trunk/src/interface/rtmididevice.cpp

Our RtMidiDevice class inherits MidiDevice (which can be realtime like
RtMidiDevice or non-real time like Midi export to file). We chose RtMidi
because the whole library is only one .cpp file and two .h files and we
compile it directly in Canorus. It supports Alsa on Linux, CoreAudio on
Mac and Winmme for Windows and it "just works". Three years ago when we
added Midi support neither PortMidi nor JackMidi were in usable state so
we didn't choose much. And TSE3 used in NoteEdit was (still is?) dead.


Regards.
