Hi folks,

As Stefan proposed I'd propose to take it either on next
Saturday in the evening or on Sunday somewhen not too early ;-)
Else I can nothing but agree concerning stability for Canorus.
I'd even prefer some of maybe "in need" features to be implemented
as plugin first so the core stability is not concerned, maybe
moving it to the core later. This way we'd also have a base concerning
the impact in performance using plugins.

It would be nice if we'd manage to split up Canorus externally
into some libraries until then - a welcome subject for the IRC
meeting. Suggestions for the meeting:

- LAC Linux Audio Conference 2009 - How will Canorus present itself
- Planned Release(s) (I'd suggest to put out at least an inbetween
   release ASAP and a second more mature one in near april)
- List of things to be done (priorities) for these releases like
   * documentation
   * translation (of the user interface mainly)
   * refactoring of the CAMainWin class - analysis and procedure
   * user interface for keyboard / midi shortcut settings
- Fill in your suggestions

I haven't checked everything what has changed in Canorus the last
weeks so something might already be finished-

Regards,

Reinhard
