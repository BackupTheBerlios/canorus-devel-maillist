The attached patch modifies the build system in two ways:

* Disables generation for 10.4.
* Disables linking against libutil under Mac.

The third change adds a new zip target to the MacOSX distribution  
process, which allows generation of a simple zip with no installers  
for people who don't want to pollute their system or have admin rights.

Actually, the reason I added this is because the pkg installer fails  
for me as a plain user (doesn't detect correctly it needs priviledges)  
and I don't trust it to run as root yet.

I'm putting my snapshots at http://personales.ya.com/gradha/ 
canorus.html. I have tried this on two machines, mine and an old ppc  
emac. On my machine the binary works but as reported at https://lists.berlios.de/pipermail/canorus-user/2009-January/000182.html 
  the ppc machine fails during launch.

I request assistance in debugging canorus for this machine. Are there  
any build switches that enforce a generation of a debug log? Maybe an  
environment variable that hardens asserts/checks? Anybody know how I  
can enable core dumping for post mortem debugging?
