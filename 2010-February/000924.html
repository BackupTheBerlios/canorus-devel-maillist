Hi Canorus Team,

I've noticed a potential bug in CAMainWin::scoreViewKeyPress().  In the
following code (lines 2097-2100 of mainwin.cpp in trunk):

        case Qt::Key_Down: {
            if (mode() == SelectMode) {    // select the upper music element
                v->selectUpMusElement();
                v->repaint();

It looks like the call should be v->selectDownMusElement(), rather than
v->selectUpMusElement().  It's not causing a problem now since neither
CAScoreView::selectUpMusElement() nor CAScoreView::selectDownMusElement()
have been implemented, but I thought I'd mention it since fixing it now
might save you time in the future.

Regards,
James Smith
