I added typesetter and PDF viewer options to settings and settings
dialog. Reinhard, I made a few small checks, if the typesetter program
exists and how it quits and connect these to a messagebox saying the
user should install lily and pdf application, if not present.

I also added CATypesetter class only having an enum of LilyPond
currently. This enum is only used for settings (which typesetter is the
default one). At first, I wanted to add this enum to CATypesetCtl, but I
thought it's more a model stuff than control. Maybe in the future
CALilypondExport etc. should inherit CATypesetter. Similar to
CAMidiExport which inherits CAMidiDevice. I'm looking for a way to
modularly "register" a typesetter into Canorus (a typesetter could also
be added by a plugin). Dunno, maybe we should remove CATypesetter, if it
brings nothing useful.
