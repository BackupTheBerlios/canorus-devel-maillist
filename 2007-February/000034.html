Is there any documentation about the file system hierarchy under Mac.
I'm a bit confused about where are the applications located - here's how
the installation works now.
To universally compile and install Canorus (no native installation or
packages), one should do:
$ cd canorus
$ cmake .
$ make
$ sudo make install

This should compile and install Canorus to default paths (ie.
/usr/local/bin&lib&share under Linux, C:\Program files\Canorus on
Windows). Where are Mac applications located?

The second way is to create a native packages (currently Debian-based
Linux and Windows). This is done by (for Windows)
$ cd canorus
$ cmake . -DCMAKE_INSTALL_PREFIX=nsis/canorus
$ make
$ make install
$ cd nsis
$ nsis setup.nsi    //creates an installation .exe for Canorus

And for debian, the installation prefix is debian/canorus/usr and
debian/rules binary is executed instead of nsis to create a .deb
package. The whole idea is to install Canorus into a local directory and
create a native package from there.

IMO, the support for Mac should be offered again genericly (by typing
make install) and to create a native package (.dmz?). This should go
into macosx/ directory.

So, my question is, which files should be installed where? As far as I
was able to see, Applications/ is used only for describing the
applications, not for their data. The core application files are still
located in /usr/bin&lib&share or maybe usr/local/bin&lib&share under Mac?
