Hi Marijn,

Thanks for your interest in Canorus.
When we were at the LAC in Berlin we discussed among many
other things a bit about that too, like having an editor widget that
can be integrated into other apps. We didn't have in mind though
just the display of score.

The main reason for that probably is that Canorus currently only
has views for "one large score in one line" (tm) ;). There are other
score editors having the possibility to render the score similar to
a printed score and we thought about offering an alternative view
if the program was designed in a way that the view can be accessed
alone via a library too.

For viewing music scores such a score view would be needed, though
it does not need to render as perfectly as lilypond does by itself (except
if we can access the lilypond renderer via library calls *daydreaming* :-)

Best,

Reinhard
