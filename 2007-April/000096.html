> These are two different things. The runtime errors are a known problem
> and may affect the operation of the toolbar menu buttons. See the
> thread "[Canorus-devel] Menu tool buttons don't work on OS X" starting
> on March 18th.
> 
> The lack of staffs and noteheads sounds like either a Python issue or a
> font issue. Since there are no staves it seems like Python did not get
> linked in properly: at startup a Python script (newdocument.py) runs to
> create staffs and put clefs in them. Try doing Insert/New sheet and
> then choose the "New Staff" button on the left (five lines with a
> star). Click on the sheet to insert a staff.
> 
> Once a staff is visible you should be able to insert clefs and notes
> using the buttons on the left. If they display properly, you should be
> all set. If they are just boxes then it's a font issue with Emmentaler,
> although as far as I know the font thing only happens on Mac.
> 
> To get staffs and clefs to display at startup, try installing Python
> 2.4 or later, along with its development files.
> 
> Sam
