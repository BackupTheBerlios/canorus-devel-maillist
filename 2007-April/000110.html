> Sounds nice. Could you give us some more information on what the code
> will actually be used for in KOffice? A notation editor integrated in

>   
A (very) rough description can be found at 
http://code.google.com/soc/kde/appinfo.html?csaid=9B5F3C70D02AA396
But what it basically comes down to is making it possible to integrate 
music in any of the koffice applications (text documents, presentations, 
but also images/pictures and even spread-sheets).
With a slightly modified canorus (and compiling it as a library), I was 
already able to load canorusml files and display them somewhat in a 
koffice application (http://eljakim.nl/~mkruisselbrink/musicflake.png).
