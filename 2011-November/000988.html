Hi Shibin,

I have added you as user to the project, so you should have access
to dotProject, the Wiki, SVN. Also you subscribed yourself to the
development list already.

I'd suggest for a start to look how our import basically works,
as our NoteEdit import is a quite important feature.

As background information you might not know:
Canorus is the successor to NoteEdit and for the 1.0 version we
aim to have at least all the features from NoteEdit.
It's a KDE3 program and slowly KDE3 is disappearing, most
distributions no longer offer so the users having still
scores created with it need to do a not-ideal inbetween step to
musicXML to import their scores currently.
NoteEdit is hosted at Berlios: 
https://developer.berlios.de/projects/noteedit

I don't remember exactly what we discussed concerning the old
parsers from NoteEdit, but the basic technique up to now is
different. Still it's of course the best source to have a look
at how NoteEdit parses it's score format basically. As the code
is still available you can check it out read-only:
svn checkout svn://svn.berlios.de/noteedit/trunk 

The core of Canorus was developed by Matevz. He's currently
working on optimizing the graphics scene of the score view
as it is quite slow at larger pieces on the end.
Also we try to find out more ways to make Canorus faster.

As you are working close to the core you probably also want
to talk to Matevz. He can help you with special questions
I might not know without studying the code ;)

Matevz, do you think that's ok or do you have anything
different in mind for him that is more important for the
next release ?

Regards,

Reinhard
