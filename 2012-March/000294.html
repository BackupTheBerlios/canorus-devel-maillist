Simplified some most common used functions.
- Renamed CAContext::findNextMusElement() -> CAContext::next().
- Renamed CAContext::findPrevMusElement() -> CAContext::previous().
- Renamed CAContext::removeMusElement() -> CAContext::remove().
- CALyricsContext::removeSyllableAtTimeStart() doesn't have autoDelete
anymore. User should always manually delete it, if needed.
CAMusElementFactory is now part of the model. No Drawable elements are
needed by configureNote() anymore.
Getter functions which doesn't simply return a property, but are slower
and generate a list or a result should have explicit 'get' before the
name property:
- Renamed CANote::chord() -> CANote::getChord()
- Renamed CAVoice::noteList() -> CAVoice::getNoteList()
Added CAPlayable::setPlayableLength().
Removed CAStaff:fixVoiceErrors().
Rewritten playable music elements manipulation:
Removed CAVoice::insertMusElement(), insertMusElementBefore(),
insertMusElementAfter, appendMusElement(), prependMusElement(),
addNoteToChord(), removeElement().
Replaced with: CAVoice::insert(), append(), remove().
Declared CAVoice::addNoteToChord() and insertMusElement() to private.
Added CAVoice::getSignList().
Added CAStaff::synchronizeVoices(). Plugin developers should now use
append(), insert() and remove() local voice functions and call this
function to synchronize shared elements among all the voices.
CAScoreViewPort::selectNextMusElement() and selectPrevMusElement() now
depend on the currently selected voice.
CAScoreViewPort::musElementsAt() now depend on the currently selected voice.

The patch was ~2500 lines long (without translations) so please test the
new stuff. Stuff to come in the next patches:
- ability to change playable length of already placed elements
- ability to insert notes "on" the rest and replace rests with them
- shift+delete shifts playable elements back instead of inserting a rest

There are also some issues with elements removal when barlines etc. are
already placed. These will be fixed when shift+delete will be implemented.
