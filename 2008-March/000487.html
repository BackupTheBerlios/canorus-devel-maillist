>  When I tried to call python scripts from pyconsole, canorus crashed.

Hi.
It is fixed now. The problem is that all python objects share the same
interpreter (all global variables). It means for example that if you
load the CLI, all output would go to the CLI.
You can now call scripts in scripts folder from the CLI.

For example you want to call script newdocument.py
Firstly, you specify the function in the script to be called (default is main)
/entryfunc newDefaultDocument
Then you call the script and pass a document reference to the function.
/callscript newdocument.py
It means, if you want to use this, the python function must look like
this: main(document).
if you write "/entryfunc", it will show you name of the entry function.
