Hi,

I managed to get the basic printing stuff running. Currently
"print" does show the pdf file produced with CAPrintCtl, so the
QPrintDialog (or similar) integration and actual sending to the
printer is still missing (but that should be relatively easily
done).

Export options for printing will (contrary to NoteEdit) via
settings. It's easier and more logical to have this all at
one place, so the user is not confused about the export
options when doing a preview of the print like in NoteEdit
(and the complicated integration of the export options to the
printer dialog can be avoided).

Concerning threads / processes: I currently don't have the
time to completely explain all, but I recommand to take a
look at the CAExternProgram class to get an idea how to
control extern programs. Could be used or adapted in a
similar way for the scripts. I'm open for discussion.
