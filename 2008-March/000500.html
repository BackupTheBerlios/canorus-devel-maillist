Ideally the goal should be that the communication part can be a class or
(for a broader "audience") maybe some C function usable by Canorus and
other applications as well (i.e. as a separate library) but used for
communication with python objects, so in the end we'd have either a "native
C++ plugin" for Canorus or you could add a wrapper class for object
oriented applications using the communication part. Ideally most of the
things would be implemented in python so the number of adaptions for the single
application is as low as possible. This way the chances are higher that the
PSF accepts your project.
