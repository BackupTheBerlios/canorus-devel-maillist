Hi Thomas,

I can't really attend FOSDEM as I have a 3 month old baby at home and 
still some exams at school to pass :)  I also won't be able to attend 
LAC this year (as it was a tradition for us and Werner for the past few 
years).

About Canorus, I'm currently speeding up the drawing code using 
QGraphicsScene and it's quite some work to do - mostly I'm rewriting all 
the drawing stuff. The rendering speed is currently very slow and it's 
priority for now. After that, we'll have a 0.8 or 0.7.5 release and I 
hope more interesting features get done after that.

btw. An idea I got into when rewriting the code. Are you and Werner 
interested in having a separated real-time engraving library? Something 
like LilyPond, but real-time using the add-modify-delete note actions 
and then returning _only_ the drawable elements which got moved/changed. 
This backend could then be used in MuscScore, Canorus and maybe some 
other apps (eg. a web service for browser note editing at server side.. 
just brainstorming :)). We had a talk on this with Werner at the last 
LAC, but he said the current engraving engine is quite complicated and 
not very clean at Muse. Maybe something worth thinking about in the future?


Regards.
-Matevž
