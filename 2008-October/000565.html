Hi all,

when doing tuplets I would like to propose a slight change of the input
policy. Let's assume we want to have the following tuplet 3/2:
    eighth eighth quarter quarter

With the mouse we can start the triplet by placing a first quarter note,
then overwrite the first note with an eighth, which could split the
quarter to eighths, which wouldn't work now, because it's not
implemented yet. But with the rests such a split is already working,
which you could try. So far the policy would be consistent.

The matter changes when I look at midi keyboard input, which is strictly
linear. Thus I propose to select, then hold fixed, the tuplet beat time
when the tuplet is selected (pressed down), so you can change the length
of the first tuplet note without interference with the tuplet beat.

I think tuplets with different time elements are not so seldom. What do
you think?

And this: I think we could restrict the selection of the tuplet
denominator to 2,4,8, because others are of no use as I can think of,
although I could be wrong.

Best Regards,
Georg
