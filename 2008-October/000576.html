Hi,
Personally, I would rather prefer some date closer to the end of the 
November to fine tune the python console. I haven't done much in Canorus 
nor ppa recently since the college is consuming nearly all my time. 
Regarding ppa, I found a way to generate corba idl files from c++ 
headers, part of it is implemented; I should try to allocate some more 
time for that (it's not vaporware I promise ;).
Reinhard mentioned some time ago idea to split Canorus into libraries, 
to modularize. I was playing with the idea and an interesting thing imho 
would be to split Canorus to 'GUI' and 'Canorus core' , two really 
independent parts. For example using the nonexistent multiprocess plugin 
system. Canorus core would run as a process (daemon) that would process 
requests from it's clients (plugins) (+ability to use events of course). 
A GUI would be also a plugin: that way more uis could be created, like 
some web interface or my favourite concept ;) canorus python console as 
fully featured user interface. Developers could test new features using 
that console without messing with the qt gui and debug them (what was a 
goal for the original python console). We're utilizing the mvc pattern 
so I guess that shouldn't be that hard. That were just some random 
thoughts for the future.
