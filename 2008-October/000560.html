Hi,

I talked with Georg about a shortcut dialog (widget) for Canorus.

I haven't decided if we should put it into the settings or make it
as separate dialog. Probably settings is enough as these are not
changed that often.

I'd make the shortcut dialog/widget more simple than in NoteEdit:
A listview or listwidget with two columns. Below are a ComboBox
and a LineEdit.

The left one contains the function to be changed (the combobox list
could contain either function without keys or all functions).
Also scripts/plugins should have assignable keys, so the list of
functions cannot be a fix array or vector.

The right one contains the shortcut. This LineEdit should be able
to automatically fetch the entered shortcut similar like in designer
Properties -> shortcut (maybe that one is no LineEdit, we would need
to check the source code what Qt is doing here actually).

The buttons should be at least delete (remove shortcut), apply/submit
to actually assign the shortcut. This way the user can cancel the action
by simply selecting a different function or closing the settings dialog.

The concept might be still rough at some points and I'd like to hear you
ideas and critics too.
