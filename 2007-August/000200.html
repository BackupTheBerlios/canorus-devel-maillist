Awesome work Matevž!!

A few issues (Tested on r555. Just noticed that r557 is out...All bugs are
still there though....)

0. Insert a chord. Select it, copy, paste.

1. Function marking seems to be a bit buggy, lots of crashes.

2. Select some notes AND some function markings, copy, select staff, paste,
crash.

3. quite funny: select some notes and some lyrics together, copy, paste. The
lyrics are pasted on the staff! No crash though.

4. I can only paste a limited number of times in a row. Sometimes on the
last time, the paste is "drawn back" and the new notes appear below the
previous notes as chords. Then paste stops working. Select a few notes,
copy, paste, paste, paste, paste,... and you'll see it.

5. Maybe related to the previous: Insert a note and a barline, select both,
copy, paste, paste, paste, (...). after the first paste, only the barline is
added.

6. Slurs are cut (removed) but not pasted.

7. Can't recall if it was like this before: the menus of menu/undo tool
buttons are separate windows, and appear in the task bar (on linux/kde at
least). I've just pressed alt+tab and saw the menu without the main window.
They should be connected somehow IMO.

8. Insert some elements, select them, copy, new document, paste, copy, crash

9. Lilypond source crashes (unless there are no elements)

10. CanorusML source just hides the vertical toolbar and nothing else (no
source is shown).

11. Viewport splitting through the window menu has stopped working long ago.
(View -> Score view works).

12. Maybe the undo-menu-tool-button-listbox (ugh..) could be a bit less
wide?
