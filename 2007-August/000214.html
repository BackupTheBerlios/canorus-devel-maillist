Short term plan:
- Canorus-0.4 beta this weekend.
- after beta - mid-september: feature freeze for 0.4, bugfixing,
translators can start working on it
- mid-september - end of september: finished translations, bugfixing
- end of september: 0.4 release

Things that got into release until now (feature freeze in mid-september!):
- Settings dialog
- Customizable colors for Score editor
- Element advanced properties dialog
- Copy/Paste
- Undo/Redo
- Splash screen
- MIDI instrument per voice support
- Key signature major/minor support and function marking integration
- Scripting engine is now always synchronized with the current API
- Python engine now uses objective Swig interface
- New CanorusML support in scripting library for directly opening/saving
Canorus files
- New playback support in scripting library
- Qt4.3 compatibility
- lots of smaller additions and even more fixes

Things that didn't get into release:
- figured bass support
- tablatures
- drum notation
- various markings like fermata, dynamics, chord, text signs etc.

I advice to fix the remaining bugs for 0.4 and implement the
non-implemented stuff in 0.5.

Longer term plan:
- release 0.5 until end of the year or maybe even later if 0.4.1 appears
to be needed
- implement as many features as possible
- VOLUNTEERS NEEDED: MusicXML import/export, translations, NoteEdit
import/export, articles and case studies on usability, requests for
features, testing and bug reporting

Known bugs needing attention:
- undo/redo bugs, see Itay's post a month ago
- when launching application and going directly to insert mode => freeze
- copy/paste bugs, see Itay's post a week ago


Regards.
- Matevž
