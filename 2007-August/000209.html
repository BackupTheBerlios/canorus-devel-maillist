Hi David!

I can confirm this now as well. The problem appeared in R540 where we
moved from non-object Canorus interface for Python to the objective one.
Now, Canorus also needs the other .py library files besides
python25.dll. So a quick solution might be to:
1) Download the latest python from www.python.org
2) Install/Extract it to eg. c:\python25
3) Copy the content c:\python25\lib to your c:\Program files\Canorus so
Canorus can find new.py, types.py and others.

The right solution would be to ship all these Python files needed by
Canorus along the installation package. The only problem I see it is
that the .py interfaces add another 3MB to our installation package and
another 13MB to the installed application.

On Linux it's much simplier as we just add package python25 as a
dependency and the package management system automagically downloads the
needed files (not to mention that python is probably preinstalled on
most distros anyway).

I added this on my TODO list and will fix it asap to make windows builds
once again useful. Thanks for the report!


Regards.
