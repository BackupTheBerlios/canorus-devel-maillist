Hi,

I implemented the settings functions for actions. I added one inline
getSingleAction function and improved generally the API of those actions.

The functions are untested but I don't expect many problems with them.
Many of them are used internally already. They are mainly needed to have
all actions of Canorus stored.

Only pointers to the actions are actually stored inside the list. I'd preferred
to use references but this is not possible due to limitation
in Qt's QAction/QObject references not being able to be compared (like
pointers can be) and assigned as those operators are private.
