I cleaned up and polished Canorus interfaces to plugins. Some things
have changed and plugin developers should not be too upset when they
find out that their plugin doesn't work anymore with the latest SVN
version of Canorus.

Plugin developers should now:
1) use "import CanorusPython" instead of _CanorusPython. They should
only use the Swig proxy classes and programm the object-way.
Example before:
Document_setTitle( myDocument, "Jingle bells" )
Now:
myDocument.setTitle( "Jingle bells" )

2) argument order is now changed in the following classes: Sheet, Staff,
LyricsContext, FunctionMarkingContext and Voice. In short, name is now
always the first argument and some other arguments now became optional.
Check the source for details. I'll upload doxygen docs to the website
when 0.4 final is out.

I already updated the newdocument.py and Harmonia plugin to work with
the latest SVN version. You can check it out.

I also removed pluginTest from the SVN. The example and its detailed
description can now be found on the wiki page "Plugins".
