Perhaps a bit OT but, regarding a heavy redesign of the API, only
recently I got my hands on Gang of Four's "Design Patterns: Elements
of Reusable Object-Oriented Software" and, besides the theory on OOP
patterns there is some good insight to design things like a wysiwyg
text editor which could be applied to a project like Canorus.

It is an old book already, and a seminal work on OOP so you probably
have already read it, but if you haven't it's definitely worth a look.
