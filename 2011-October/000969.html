Hi Matevž,

Concerning Mercurial: I already considered to move to that somewhen
so if this is the opportunity let's do it. I still don't know much about
it except concepts but Practice makes perfect.
Integration into eclipse is easy.

For Ubuntu launchpad is great, don't know if debian folks use it too
for custom packaging but that shouldn't be a big deal anyways. Maybe
I can see what's the problem with canorus build.

For SuSE maybe pacman offers possibilities to build custom packages
(or maybe there is an opensuse buildserver). Georg should know more about
that.

There is a fedora buildsystem available but I don't know anything
about it. Maybe someone can shed a light in here.

Other projects like Mageia/Mandriva sometimes offers their own
buildservice. Though I like Mageia a lot I think we should concentrate
on the above, probably not more than 2-4 binaries and mainly for
releases (i.e. nightly builds on request).

I don't know the possibilities for the other non-Unix systems. Any
suggestions are welcome, I'd like to avoid depending on individual
persons to build (at least) releases.

Yeah it's unfortunate that Savannah/Alioth offer no wiki. The only
other possibility might be separate the project wiki from the
project itself (like on wikihost.org).
Any more opinions for project hosting ?

Regards,

Reinhard
