The biggest feature I planned for 0.8 is rewrite of the graphic engine. 
This is in graphicsscene branch already, so it's easy to publish the 
current trunk as 0.7.1. If you ask me, this is also the right way to do. 
If we have a bigger feature, we introduce new branch for that specific 
feature and then merge it back to trunk, when it's done. This way we can 
publish new releases meanwhile. This is also how hg works (own branch 
for each feature or bug).
