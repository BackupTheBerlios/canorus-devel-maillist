Hi Nils,

Thanks for the feedback. It's not easy to decide where to host the project.
Sourceforge is fully-feature-blown but slow at times. Github lacks in some
parts (shell access, forum, mailing list, mercurial), so the danger is that
the project will be split at too many pages. Maybe we choose two hoster, one
for project-related and another for web/communication-related things.

Do you have any suggestion for a webhoster supporting MediaWiki, Mailing
Lists and/or forums ? We still could use sourceforge for that. From my
experience it's easy to integrate an external access to the code itself.

Regards,

Reinhard
