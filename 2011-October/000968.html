Hi Reinhard.

I vote for sourceforge. I already host a few projects there and they also
offer wiki out of the box, so we don't bother with building a web page from
scratch but just use the old one.

For the ubuntu nightlies, I prefer launchpad. I already made a Canorus
project there which imports berlios trunk for now and tries to build the
packages. See https://launchpad.net/canorus
We should move from berlios svn now to sourceforge, preferably Mercurial and
then set the launchpad to import sources from there.

For the nightly packages, unfortunately I haven't been able to create a
working build script yet, so no working .deb packages are generated on
Launchpad :/
