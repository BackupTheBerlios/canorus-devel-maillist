Hi,

I'd like to know your opinion on bugfix releases. During 0.7 period
and a long time with no release I often wanted to give away a new
release with mainly problems fixed for the users.

This of course introduces some additional work usually:
- Fix problems on the stable branch or backport from trunk
- Merge the fix to the trunk (if not already fixed there)
- Release Management work increases (with revisions like 0.8.1)

Some parts will be eased when moving to Mercurial making it much
easier to merge (as it was developed for the merging task).

Also build server(s) would make it easier to not only check that
everything compiles correctly but also the additional binary
releases. I aim to have rpm, deb, Windows exe and a Mac binary.

I'd prefer to introduce it now (after 0.8 release) instead of
waiting until 1.0.
What do you think ?
