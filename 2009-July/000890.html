Hi Georg & Itay,

I opt for introducing a debug and release version as it usually is
problematic debugging optimized code.

Default build should be with optimization (never optimize higher
than -O2 !). Also debug code can lead to slower performance to. Did
you test that beforehand ?

The only bad thing about this is for package managers. The usual
way is to build with debug option and to introduce separate debug
packages (concerns mainly future canorus libs) and the release
will be strip(ped) with the "-g" option.

any other ideas ?

Regards,

Reinhard
