Hi Reinhard.

The problem with fonts was specific to Mac OS X AFAIK. Ubuntu uses Qt 
4.5.0 and works fine.

About the release - I didn't have time to change any internals yet 
(barlines and other shared elements part of the staffs instead of the 
voices). But I looked into faster rendering (using QGraphicsView), but 
haven't committed any code yet.

Maybe we should release version 0.7.5 at the end of the summer. 
Noteworthy features and improvements for the release:
- Import and export:
 - Vastly improved Midi import using PMidi backend
 - Export of articulation marks, fingering, dynamic marks etc. to Lilypond
 - Improved MusicXml import (instrument and midi channel, movement-title)
- Improved GUI and user experience:
 - Automatic animated scroll while playback
 - Progress bar when loading (TODO for saving)
 - Tab and Shift+tab for next and previous voice
 - Faster score rendering (TODO)
 - Keyboard and Midi shortcuts (partly TODO)
- Updated RtMidi library
- Internal cleanups

Btw. I commited a few fixes to the Windows version of Canorus. I can 
confirm that on Vista it builds and installs now, fonts work fine, print 
preview and export to PDF using LilyPond works out of the box, if Lily 
is installed. So it's usable on Windows.


Regards.
-Matevž
