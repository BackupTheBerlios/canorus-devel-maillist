I think in the future we should move the shared signs outside the voice 
(to CAStaff directly). And I wouldn't use CAVoiceCtl and CAStaffCtl 
classes but put all the control stuff into the CAVoice and CAStaff 
itself (not many public control functions will exist I think - maybe 5, 
others are more model for insertion/deletion). This way we would 
simplify the score operations to the core classes CAVoice, CAStaff etc. 
No special control classes would be needed for the Python developer.

I would create class CAQuantization for the tuplet and other quantiazion 
operations. I would create CAMainWinMode, CAMainWinInsert, 
CAMainWinDelete, CAMainWinEdit for the GUI operations specific to the 
current mode.

I need Reinhard's opinion on this though.
