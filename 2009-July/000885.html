Hi all,

midi import now builds chords where they're easily detectable, which
seems to help a little in the speed issue, probably because chords seem
to be a little more efficient then single notes. Next I'll maybe look
into tuplet recognition. Just a few remarks:

At display, the stems of chord members should have the same direction
regardless of the individual stem settings.

At display, tempo settings should have and follow an on/off toggle,
because they clutter the score heavily. And many people like the
"humanized" playback. For velocity, which is not yet imported, that's
the same.

If you have a midi file that doesn't get imported I would look into
that. Mine do all get imported, we just have the speed issue for bigger
scores (which can be solved, I believe).

Maybe we can close some bugs on berlios?

Editing functions (or parts) in ui/mainwin.cpp that don't use xy-coords
any more should be moved to staff.cpp or voice.cpp, to reduce mainwin
and to avoid duplicates of edit functions.

I wouldn't hold back features because of translation set backs.

So, Best Regards,
Georg
