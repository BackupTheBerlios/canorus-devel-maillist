Hi Matevž,

The user docs generation has already used a Makefile (which requires 'sh' at
least), and since updating qhp/qhcp is optional (you can generate the docs
with the files currently in SVN), I didn't think a bash dependency would
really be a problem. FWIW, the script could be run on Windows using either
Cygwin or MSYS.

That said, I wrote this script mostly as a bash exercise, so feel free to
replace it with something else. Porting to CMake would probably require some
work.
