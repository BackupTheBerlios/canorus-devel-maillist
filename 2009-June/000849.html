Hi Matevž,

the differentiation (offset) is done in CAMidiExport::send(). 
CARtMidiDevice::send() doesn't use this time parameter at all. Like in
sendMetaEvent() I preferred to do these midi qirks in CAMidiExport,
because it's only needed there.

timeLength is a name I got from
CAPlayableLength::playableLengthToTimeLength(), and it means a time
scaled like note values. Maybe time would be the right name, like it is
all around in canorus.
