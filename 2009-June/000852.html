Hi Georg,

According to the current concept of document the behaviour is logical as
Opening is only for documents and not for sheets.
If 130.xml were a multi-sheet document you would not be able to load it
into a separate sheet.

I see some possibilites (code-wise) how to solve this problem:
- Append document (adds all sheets after the current selected sheet)
- Insert document (same but before the current selected sheet)
- Load/Save sheet, xml file containing only one sheet

I'd say all the above ideas are useful to be implemented. I'm not quite
sure how to handle sheets, as we already have .xml and .can. On the one
hand for the user it's easier to differentiate these two file "types"
On the other hand sheet should only be saved as single-sheet document
to spare work, so there is in practice no real difference (you could
even load single sheet as documents without problems of course).

Regards,

Reinhard

> Hi, here's the recipe:
>
> When you start canorus, you can open 130.xml from the example directory,
> for example. Then, with Window->New, this score will be overwritten by
> the new one, so instead, we say, Insert->New Sheet. This is okay, and
> editing of the two sheets is now possible.
>
> But when we want to read into the new sheet an already available score,
> for example 130.done.xml, we activate the new sheet, say File->Open. A
> window
>      Unsaved changes
>
>      Document "Untitled" was modified. Do you want to save the changes?
>      Yes   No   Cancel
> appears. You have not many options, let's say we say No. In the file
> dialog then we select 130.done.xml, and the second sheet disappears, and
> 130.done.xml will be the only score.
>
>
> So no cross-pollination between scores,
