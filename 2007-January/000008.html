Hi, 

Canorus rev 324 gives me the following errors when started:

...
QMetaObject::connectSlotsByName: No matching signal for 
on_actionAbout_Canorus_activated()
Object::connect: No such signal QAction::activated()
Object::connect:  (receiver name: 'MainWindow')

As a consequence no menuitem works. 

This is because QAction doesn't have a activated Signal.
(http://doc.trolltech.com/4.2/qaction.html)
In QT3 it had but I guess now you should simply use the "triggered" signal. 
So the methods must be named on_actionAbout_Canorus_triggered().

When I simply cange every activated in triggered in mainwindow.cpp/.h
everytihng works again.
Does it work for anybody without that change. Maybe its only because i 
compiled QT without QT3 support. But even then triggered would be the right 
way.
