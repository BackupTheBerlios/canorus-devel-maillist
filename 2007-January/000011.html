Hi,

I've been working the last two days hard and brought the long-awaited
basic plugin<->UI connectivity. Users can now finally add a custom menu
entry to the menubar and an import/export filter to the import/export
Canorus dialog.

In short what I have done is that plugin now consists of various action
tags, menu tags and/or toolbar tags. Localization is also supported for
UI strings.

== Plugin action ==
Plugin action works similar to QAction. Apart from that, it has basic
scripting properties:
- name (probably the same as Qt's objectName - still thinking about it)
- programming language
- filename
- function
- needed arguments
- is GUI rebuild needed after execution
When called, it executes arbitrary external function in a script.

Plugin action brings also the second part along - the aditional
properties which tell when is the action triggered:
- import filter (action activated when file->export and file is selected)
- export filter (action activated when file->import and file is selected)
- parent-menu or toolbar item (action activated when a menu or toolbar
item clicked)
- internal Canoruson-action (action activated on one of the built-in
Canorus events eg. onScoreViewPortClick)
Actions are defined in <action> stanza in canorusplugin descriptor xml file.

== Plugin menu ==
Plugin menu works similar to QMenu. It consists of various actions (ie.
menu items).
If parent-menu is set, then the menu is added to the parent menu object.
If not, it's added to the top-level toolbar of the main window.
Plugin menu is defined inside the <menu> stanza in canorusplugin
descriptor xml file.

== Plugin toolbar ==
Should be similar to menu - a placeholder for various actions. Gonna be
implemented when needed.

I attached the example canorusplugin.xml descriptor file with some
explenatory comments. I also committed a test plugin to SVN now which
adds a simple export filter and a harmony analysis tool menu. It doesn't
do anything though.
