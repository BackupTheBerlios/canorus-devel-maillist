Matevž Jekovec wrote:
> I'm currently in designing how to implement the auto bar stuff. I'd do
> something like when you insert a new note, it determines the current
> time signature and the last barline and if the timestart of the new note
> and the last barline is too big, it adds a barline in front of the new note.
>   
This would be a good. Something to consider is the case when a user 
enters a note that can't fit in the current measure. For example, in 4/4 
time: a quarter note followed by two half notes. In this scenario some 
scorewriters will convert the last half note to a quarter note tied over 
the bar line to a quarter note of the next measure.
