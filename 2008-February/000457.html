Well the main problem is currently that (AFAIK) I cannot export Qt objects.
But this is the main requirement so that this class works. I considered for
a different class to have a thin wrapper, so that mainly all Qt specific
stuff is hidden. The main problem however is how to signal a script that
something changed ? The solution is of course a observer pattern, i.e. the
script tells me that it waits for input (this is similar to signal/slot but
independent of it and only on C++ layer). There are both solution for C++
native (like sigslot) and python available, but the main question is how
to combine the both (and use swig).

Signals in sigslots have a template syntax, like mysignal<int>. I don't
know much about swig to really be able to tell you if this works together
with python or ruby.

Except the class calling an extern program I also want to offer a generic
dialog / form display class. For this class I planned the possibility to
use it with scripts. This could be a similar wrapper like above. Probably
the best is to have a base class (that can be used standalone for simple
interfaces) for the signal / slot functionality.
