Depends on what you want to do. Debugging is switched on currently by
default - see line 10 in trunk/src/CMakeLists.txt, switch -g passed to
gcc. You can debug it using gdb for example.

If you want to view the state of the score runtime using python, Štefan
did a great job by making a python CLI console inside Canorus. There is
a global variable 'document' which represents the current document and
you can analyse it then.
