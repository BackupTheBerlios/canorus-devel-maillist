Hi,

I did no changes to the Preview function, it still works and if your
system pdf viewer supports printing you can directly print from there
as it uses the original pdf file from lilypond.

Printing works too but is limited to single page documents due to a
bug or restriction from the QSvgRenderer - it seems to render only
the whole document at once and so the pages are all on one page instead
of several ones. Unfortunately there is no method to select one page.

So for printing I'd currently recommand to use the first method -
preview and print from your pdf viewer.

For the multiple pages pdf we could manually split the svg files into
several pages. But according to the lilypond manual (I use the latest
dev version 2.11.55) it should create already several svg's by itself.
Matevz already asked on the lilypond list for a solution to this problem.
This way I could render all the svg files to the QPrinter object.
