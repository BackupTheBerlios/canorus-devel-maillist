Transposition is not yet implemented, but should not be a problem. The
backend is already there - you could take a look at CADiatonicPitch
(note name + accs) and CAInterval (prime, second, perfect, major etc.).
Every note has a diatonic pitch value set. And CADiatonicPitch has +
operator implemented for the interval. So you can do existing pitch +
interval = new pitch and apply the new pitch to all the selected notes.
Heh, we even have the + defined for the intervals (eg. prime + third =
third).

The most interesting thing would actually be to provide a GUI for the
transposition. Also, what happens to the key signature.

This is on our TODO list for version 0.6 anyway. If you're interested
in, you can contribute this feature!
